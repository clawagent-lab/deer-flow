# 思考块测试指南

## 快速测试

### 方法 1：使用模拟数据

1. 在浏览器中访问应用并添加 `?mock=reasoning-example` 参数（如 `http://localhost:3000?mock=reasoning-example`）
2. 发送任意消息
3. 观察计划卡片上方是否出现思考块

### 方法 2：启用深度思考模式

1. 确保配置了 reasoning 模型（如 DeepSeek R1）
2. 在聊天界面点击「Deep Thinking」按钮
3. 发送一个需要规划的问题
4. 观察是否出现思考块

## 测试场景

### 场景 1：完整的深度思考流程

**步骤**：
1. 启用深度思考模式
2. 发送问题，例如「什么是 vibe coding？」
3. 观察完整交互流程

**预期行为**：

#### 阶段 1：深度思考开始
- 思考块立即出现并展开
- 使用 primary 主题色（边框、背景、图标、文字）
- 显示加载动画
- 不显示计划卡片
- 推理内容实时流式更新

#### 阶段 2：思考过程中
- 思考块保持展开状态
- primary 主题色持续显示
- 推理内容持续增加
- 加载动画持续显示
- 计划卡片仍然不显示

#### 阶段 3：开始接收计划内容
- 思考块自动折叠
- 主题从 primary 切换为默认
- 加载动画消失
- 计划卡片以入场动画出现（opacity 0→1，y 20→0）
- 计划内容保持流式更新效果

#### 阶段 4：计划流式输出
- 标题逐步显示
- 思路内容流式更新
- 步骤列表逐项显示

#### 阶段 5：计划完成
- 思考块保持折叠状态
- 计划卡片完全显示
- 用户可手动展开思考块查看推理过程

### 场景 2：手动交互测试

**步骤**：
1. 在思考完成后，手动点击思考块标题栏
2. 验证展开/折叠功能

**预期行为**：
- 点击可正常展开/折叠
- 动画效果流畅
- 内容完整显示
- 不影响计划卡片显示

### 场景 3：边界情况测试

#### 3.1 只有推理内容，没有计划内容
**预期**：思考块保持展开，不显示计划卡片

#### 3.2 没有推理内容，只有计划内容
**预期**：不显示思考块，直接显示计划卡片

#### 3.3 推理内容为空
**预期**：不显示思考块，直接显示计划卡片（`content.trim() === ""` 时组件返回 `null`）

## 预期外观

- 深度思考开始时自动展开显示
- 思考阶段使用 primary 主题色（边框、背景、文字、图标）
- 带有 18px `Lightbulb`（灯泡）图标
- 标题文案为翻译键 `chat.research.deepThinking`（中文「深度思考」）
- 使用 `font-semibold` 字体权重，与 CardTitle 保持一致
- `rounded-xl` 圆角设计，与其他卡片组件统一
- 标准的 `px-6 py-4` 内边距

## 故障排除

### 思考块不显示
1. 检查消息是否包含 `reasoningContent` 字段
2. 确认 `reasoning_content` 事件是否正确处理
3. 验证消息合并逻辑（`mergeTextMessage`）是否正常工作

### 思考块不自动折叠
1. 检查 `hasMainContent` 逻辑
2. 验证 `useEffect` 依赖项
3. 确认 `hasAutoCollapsed` 状态管理

### 计划卡片显示时机错误
1. 检查 `shouldShowPlan` 计算逻辑
2. 验证 `isThinking` 状态判断
3. 确认消息内容解析正确

### 主题切换异常
1. 检查 `isStreaming`（实为 `isThinking`）状态传入
2. 验证 CSS 类名应用
3. 确认条件渲染逻辑

### 内容显示异常
1. 检查 Markdown 渲染是否正常
2. 确认 CSS 样式是否正确加载
3. 验证动画效果是否启用

### 流式传输问题
1. 检查 SSE 连接状态（前端通过 SSE 而非 WebSocket 接收事件）
2. 确认事件流格式是否正确
3. 验证消息更新逻辑
